# 施策管理 Web アプリ 仕様書

## 1. 概要

### 1.1 目的

社内の **施策（プロジェクト・改善案）** を一覧・詳細で管理する。依頼者・担当・優先度・ステータス・説明文・工期・備考などを一元化し、**コメントで経緯を残す**。

### 1.2 利用想定

- **端末:** PC / スマートフォン / タブレット（レスポンシブ）
- **認証:** なし（単一環境・社内共有を想定）
- **UI:** 明るいニュートラル背景・白いカード・システムブルー（`#1a73e8`）アクセント（Apple / Google 純正アプリに近いトーン）

### 1.3 リポジトリ・本番 URL（参考）

- GitHub: `hmatsuo-ai/sunhome-sisaku`
- 本番例: `https://sunhome-sisaku.vercel.app/`（実際の URL は Vercel の Production ドメインに従う）

---

## 2. 機能一覧

| 機能 | 説明 |
|------|------|
| 施策一覧 | 左ペインに施策名のみリスト表示。ステータスを色付きドットで表示 |
| 施策詳細 | 選択した施策の全フィールドを表示。コメント一覧・投稿 |
| 新規作成 | 全入力項目を登録。**空欄のまま保存可能**（未入力は空文字として保存） |
| 編集 | 既存施策を更新。**最終更新日（`updatedAt`）は自動更新** |
| コメント | 施策ごとに本文（必須）・投稿者名（任意）・投稿日時（自動） |
| モバイル表示 | 詳細表示中は左一覧を非表示。「一覧に戻る」で一覧へ |

## 3. 画面仕様

### 3.1 ホーム（`/`）

- **ヘッダ:** タイトル「施策管理Webアプリ」
- **左ペイン（一覧）**
  - 見出し「施策一覧」、ボタン「新規作成」→ `/policies/new`
  - 各行: ステータス色・施策名（空の場合は「（名称未設定）」）・ステータスラベル
  - 行タップで当該施策を選択し詳細表示（モバイルでは一覧を隠す）
- **右ペイン（詳細）**
  - 施策未選択時: 案内文と「新規施策を作成」リンク
  - 選択時: 詳細カード＋コメント欄

**ステータス色（一覧ドット）**

| ステータス | 色のイメージ |
|------------|----------------|
| 未着手 | グレー |
| 進行中 | ブルー（Google ブルー系） |
| 完了 | グリーン |

### 3.2 新規作成（`/policies/new`）

- 全フィールドのフォーム
- 「一覧へ戻る」→ `/`
- 保存成功後 → `/`

### 3.3 編集（`/policies/[id]/edit`）

- 既存値を埋めたフォーム
- 「一覧へ戻る」→ `/`
- 更新成功後 → `/`

### 3.4 表示上の空欄

詳細の各項目が空のときは **「—」**（ダッシュ）を表示。

## 4. データモデル（DB）

**RDB:** PostgreSQL（本番は [Neon](https://neon.tech) 等、Vercel Marketplace 連携を想定）  
**ORM:** Prisma

### 4.1 Policy（施策）

| カラム | 型 | 説明 |
|--------|-----|------|
| `id` | Int PK | 自動採番 |
| `name` | String | 施策の名前 |
| `requesterName` | String | 依頼者の名前 |
| `ownerName` | String | 担当者 |
| `priority` | Enum | `HIGH` / `MEDIUM` / `LOW`（表示: 高・中・低） |
| `status` | Enum | `TODO` / `IN_PROGRESS` / `DONE`（表示: 未着手・進行中・完了） |
| `pastIssues` | Text | これまでの問題点 |
| `systemDescription` | Text | システムの内容 |
| `effect` | Text | 施策による効果 |
| `estimatedDuration` | String | 予想工期（自由記述） |
| `actualDuration` | String | 実際の工期（自由記述） |
| `notes` | Text | 備考 |
| `createdAt` | DateTime | 作成日時（自動） |
| `updatedAt` | DateTime | 最終更新日（更新時自動） |

**バリデーション（アプリ側）**

- 文字列フィールドは **未入力可**（API で正規化し空文字として保存）
- `priority` / `status` が不正または欠落時は既定値（中・未着手）

### 4.2 Comment（コメント）

| カラム | 型 | 説明 |
|--------|-----|------|
| `id` | Int PK | 自動採番 |
| `policyId` | Int FK | `Policy.id`（削除時 CASCADE） |
| `body` | Text | 本文（API で必須） |
| `authorName` | String? | 投稿者名（任意） |
| `createdAt` | DateTime | 投稿日時（自動） |

## 5. API

ベースパス: アプリオリジン直下（例: `https://example.com/api/...`）

### 5.1 施策

| メソッド | パス | 説明 |
|----------|------|------|
| `GET` | `/api/policies` | 施策一覧（`updatedAt` 降順） |
| `POST` | `/api/policies` | 施策作成（JSON ボディ） |
| `PUT` | `/api/policies` | 施策更新（ボディに `id` 必須） |

### 5.2 施策1件

| メソッド | パス | 説明 |
|----------|------|------|
| `GET` | `/api/policies/[id]` | 施策詳細 |

### 5.3 コメント

| メソッド | パス | 説明 |
|----------|------|------|
| `GET` | `/api/policies/[id]/comments` | コメント一覧（`createdAt` 降順） |
| `POST` | `/api/policies/[id]/comments` | コメント追加。`body` 必須、`authorName` 任意 |

## 6. 環境変数・ビルド

### 6.1 必須（本番・開発共通）

| 変数名 | 用途 |
|--------|------|
| `DATABASE_URL` | Prisma 接続（プール URL 推奨） |
| `DATABASE_URL_UNPOOLED` | マイグレーション用直結 URL |

### 6.2 ビルド（`package.json`）

- `npm run build` = `prisma migrate deploy` → `prisma generate` → `next build`

### 6.3 Vercel

- **Framework Preset:** `Next.js`（`vercel.json` で `framework: nextjs` も指定可）
- Neon 連携時は上記 DB 用変数が Production / Preview に揃っていること

詳細なトラブルシュートは [README.md](../README.md) を参照。

---

## 7. 非機能・制約

- **認証・権限:** なし。誰でも一覧・編集・コメント可能。
- **同時編集:** 楽観的ロックなし（後勝ち）。
- **ファイル添付:** なし。
